# Privacy Gotchas

Use this file when the user is making a real send or receive decision and needs the practical caveats, not theory.

## Common support warnings

- wallet support and protocol privacy are not the same thing
- address compatibility can shape what privacy properties are available in practice
- sending behavior can leak more than users expect if they assume all paths are equivalent
- exchange or third-party wallet support may lag the protocol story

## Good support phrasing

- "This depends on what your wallet and counterparty actually support."
- "Do not assume privacy is automatic just because the network supports shielded transfers."
