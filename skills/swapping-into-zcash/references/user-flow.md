# User Flow

Use this file when the user is asking how to swap into Zcash rather than how to implement the infrastructure.

## Standard support flow

1. identify the source asset and chain
2. identify the destination form the user actually wants: exchange balance, transparent ZEC, or shielded ZEC-capable wallet flow
3. explain the quote and deposit step
4. explain what the user should watch during execution
5. explain how to verify arrival and what a refund path looks like

## Good support phrasing

- "This is a cross-chain swap flow, not a single native Zcash transaction."
- "Your privacy properties change across the different legs of the flow."
- "Verify the destination wallet and support path before depositing."
