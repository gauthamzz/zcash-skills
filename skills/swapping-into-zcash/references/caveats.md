# Caveats

Use this file for the warnings that should be explicit in any serious answer about swapping into Zcash.

## Operational caveats

- destination support matters: not every wallet, service, or exchange path supports the same Zcash receive modes
- quotes or deposit instructions may expire
- incomplete deposits can produce unexpected outcomes
- refund paths must be explained before the user sends funds

## Privacy caveats

- a cross-chain swap into Zcash is not private end-to-end just because the destination asset is ZEC
- the source-chain leg, the execution provider, and the destination wallet can each expose different information
- if the user wants shielded privacy, say where that property begins and what happened before that point

## Support caveats

- do not promise the swap provider, wallet, or counterparty path supports shielded delivery unless verified
- do not use "bridge" and "swap" interchangeably without clarifying the mechanism
