# Routing Guide

Use this file when a request touches more than one Zcash domain and the next skill is not obvious.

## Common routes

- "Explain Zcash" -> `zcash-beginners`
- "Define Orchard / Sapling / nullifier" -> `zcash-glossary`
- "Which wallet should I use?" -> `zcash-wallet-guide`
- "How does shielding actually work?" -> `zcash-protocol-explainer`
- "Why is my node stuck?" -> `zcash-node-ops`
- "Which RPC should I call?" -> `zcash-rpc-reference`
- "Who builds what in Zcash?" -> `zcash-ecosystem-map`

## Blended requests

- beginner + wallet: explain the concept briefly, then route to wallet guidance
- protocol + command: explain at high level, then route to RPC guidance
- ecosystem + governance: keep facts and opinions clearly separated
