# User Playbook

Use this file when the user wants a practical answer like "I have SOL in wallet A and want funds to arrive in wallet B more privately."

## Support sequence

1. identify source chain and asset
2. identify destination chain and destination wallet
3. ask whether the user wants to keep value in ZEC for a while or immediately exit
4. explain the route in legs
5. explain what to verify before each leg

## Good answer shape

1. source-chain action
2. Zcash entry point
3. shielded leg
4. destination leg
5. caveats and failure points

## Key support language

- "The privacy leg begins only once the value is actually in a shielded Zcash flow."
- "Your Solana or Base send into the route is still visible on that source chain."
- "If you exit back to another public chain, that exit creates a new visible leg."
