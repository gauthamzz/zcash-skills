# Zcash Glossary

Define terms fast. Keep distinctions intact.

## What this skill does

- explain Zcash terms in plain language
- separate similar but non-identical concepts
- keep answers short unless the user asks for deeper mechanics

## Example requests

- "What is Orchard?"
- "What is a nullifier in Zcash?"
- "Unified address vs receiver?"

## Working style

1. Define the term in one or two sentences.
2. Add one contrast when confusion is common.
3. Route deeper questions to `../zcash-protocol-explainer/` or `../zcash-wallet-guide/`.

## Guardrails

- Do not flatten historical and current terms into the same thing.
- Do not assume the user wants cryptographic depth.
- Do not use a glossary answer to sneak in unrelated opinion.

## References

- Read `references/terms.md` for the core terms and the most important contrasts.
